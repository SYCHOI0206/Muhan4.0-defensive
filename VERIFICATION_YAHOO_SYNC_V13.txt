SOXL V4 방어형 v13 Yahoo Finance 동기화 검증

판정
- Yahoo Finance 웹은 많은 거래소의 실시간 스트리밍 시세를 제공한다.
- 그러나 정적 GitHub Pages에서 사용할 수 있는 공식 공개 Yahoo API는 없다.
- query1/query2 chart endpoint는 기술적으로 조회 가능하지만 비공식이며 CORS 및 호출 제한 가능성이 있다.
- 따라서 계산기의 기본 경로는 GitHub Actions 서버가 Yahoo를 조회하고 같은 저장소 JSON으로 제공하는 방식이다.

v13 변경
1. 사용자가 업로드한 CSV로 만든 종가 이력을 제거했다.
2. data/close-history.json은 빈 Yahoo 동기화 저장소로 시작한다.
3. 최초 배포 또는 수동 full refresh 시 Yahoo에서 SOXL 전체 일봉을 다시 구축한다.
4. 평일 미국 동부시간 16:20, 17:05에 자동 동기화한다.
5. 당일 장중 미완성 일봉은 확정 종가로 저장하지 않는다.
6. 선택한 날짜와 정확히 일치하는 종가만 입력한다.
7. 브라우저 Yahoo 직접 조회는 보조 경로일 뿐이며 실패해도 Actions 동기화가 기본 경로다.
8. 주말/휴장일에는 다른 날짜 종가를 대신 넣지 않는다.
9. 전략 핵심 엔진은 변경하지 않았다.

초기 사용
- GitHub Actions > Sync Yahoo SOXL prices > Run workflow
- full_refresh=true로 한 번 실행
- 이후 평일 자동 동기화
