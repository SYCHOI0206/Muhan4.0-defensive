5. GitHub Pages 반영 뒤 설치형 앱을 완전히 종료하고 다시 실행합니다.
6. 옛 화면이 남으면 사이트 데이터/설치 앱 캐시를 지우거나 재설치합니다.

공식 계산 규칙
1. 정상모드
- 별지점 22%, 전량목표 22%, 쿼터매도 1/3
- 당일 시작 T·현금·평단으로 전반/후반, 별지점, 1회매수금을 고정
- 정수주 floor, 수수료·세금·슬리피지 미반영

2. -22.5% 급락 방어
- 일간 종가수익률이 -22.5% 이하이고 정상매수가 실제 체결되면
  다음 거래일의 정상매수를 0.25T 단일 LOC로 대체
- 전반: 별지점 LOC와 평단가 LOC 중 높은 가격
- 후반: 별지점 LOC
- 매수예산: 당일 시작 1회매수금 × 0.25
- 체결 시 T 증가: +0.25T
- 미체결 시 당일 소멸
- 전량매도 또는 리버스모드에서는 예약 취소

3. 리버스 매도차수별 고정금 버킷
- 리버스 첫날 보유수량의 10% 매도
- 첫 버킷 원금: 첫 리버스 매도 직후의 전체 현금
  (리버스 진입 전에 남아 있던 현금 + 첫 매도대금)
- 이후 리버스 매도 때마다 해당 매도대금으로 새 독립 버킷 생성
- 리버스 매수 체결금은 활성 버킷의 당일 요청액 비율대로 차감
- 버킷별 미사용 금액은 남지만, 당일 미체결 예산을 다음 날 한도에 가산하지 않음
- 리버스 탈출 후 일반모드 복귀 시 버킷 회계상태 초기화

4. SOXL V4 방어형 리버스 매수
- 당일 종가를 포함한 MA200 사용
- MA200 데이터 200일 미만: 각 버킷 원금의 25%
- 종가 >= MA200: 각 버킷 원금의 15%
- MA200×0.85 <= 종가 < MA200: 각 버킷 원금의 10%
- 종가 < MA200×0.85: 각 버킷 원금의 5%
- 각 버킷 요청액 = min(버킷 잔액, 버킷 원금 × 당일 비율)
- 전체 매수예산 = 각 버킷 요청액 합계와 현재 현금 중 작은 금액
- 리버스 매수 체결 시 T는 원문 리버스와 동일하게
  T + (20-T)×당일 MA200 매수비율로 증가

5. 자동 리버스 처리순서
- 탈출: 종가 > 평단×0.80 → 당일 거래 없이 다음 거래일 일반모드 복귀
- 매도: 종가 >= 직전 5거래일 종가평균 → 보유수량 10% 매도, 새 버킷 생성
- 매수: 종가 <= 직전 5거래일 종가평균-0.01 → MA200 비율로 버킷 매수
- 위 조건에 해당하지 않으면 거래 없음
