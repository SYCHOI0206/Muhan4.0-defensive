SOXL V4 방어형 v18 입출금·사이클·자산분석 검증

구조
- v40_defensive_core.js 변경 없음
- v40_defensive_patch.js에는 분석 모듈이 수동기록을 같은 방식으로 재생할 수 있도록 읽기용 브리지 3개만 노출
- 신규 v40_portfolio_analytics.js가 입출금 원장, 통합 재생, 사이클 요약, 자산 그래프를 담당

입출금
- 설정 탭에서 입금/출금, 날짜, 거래 전/후, 금액, 메모를 기록
- 입금은 현금과 순투자원금을 같은 금액만큼 증가
- 출금은 현금과 순투자원금을 같은 금액만큼 감소
- 출금액이 해당 시점 현금을 초과하면 저장 차단
- 초기원금 수정은 기존과 같이 가능하나 기록이 있으면 과거 전체 재계산 경고

성과 탭
- 총자산 = 현금 + 보유수량×최근 기록 종가
- 순투자원금 = 초기원금 + 누적입금 - 누적출금
- 순수 투자손익 = 총자산 - 순투자원금
- 총자산/순투자원금/순수손익 SVG 그래프
- 전체/1년/6개월/현재 사이클 기간 필터
- 전량매도 기준 완료 사이클 자동 요약 및 진행 중 사이클 표시
- 입출금 및 사이클 CSV 내보내기

호환성
- 기존 백업은 cashFlows=[]로 자동 변환
- JSON 백업에는 cashFlows와 schemaVersion=18이 자동 포함
- 입출금이 없을 때 기존 v17 replay와 동일 상태가 나와야 함
- 서비스워커 캐시 v18, 신규 분석 JS 포함

독립 로직 테스트
- 입출금 0건: 기존 replay와 cash/shares/avg/T/cycle/mode 상태 동일
- 입금 $5,000 및 출금 $1,000: 순투자원금 $14,000으로 계산
- 순수손익 = 총자산 - 순투자원금 일치
- 당시 현금을 초과하는 출금은 저장 차단
- 테스트 시나리오에서 완료 사이클 1건 자동 생성
- JavaScript 문법검사 통과
